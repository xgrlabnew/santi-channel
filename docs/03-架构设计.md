# Santi Channel 架构设计

> **⚠️ 注意：本文档描述的是完整版本的架构设计，当前MVP版本已大幅简化。**
> 
> **MVP版本架构**：
> - sChat: 极简Web客户端（纯HTML/JS/CSS）
> - Santi Channel: 单文件Node.js插件（WebSocket转发）
> - 无数据库，无用户系统，无复杂状态管理
> 
> **简化原则**：
> 1. 最少代码，最少依赖
> 2. 先有可工作的版本
> 3. 逐步添加功能
> 
> **当前实现**：参考 `santi-channel-mvp/` 目录中的实际代码

## 🔧 技术栈选择

### 后端技术栈
| 组件 | 技术选择 | 理由 |
|------|----------|------|
| **运行时** | Node.js 18+ | 异步I/O、生态丰富、性能优秀 |
| **语言** | TypeScript 5+ | 类型安全、开发效率高、维护性好 |
| **Web框架** | Express.js + Socket.IO | 轻量级、成熟稳定、社区活跃 |
| **数据库** | PostgreSQL + Redis | 关系型+缓存，功能全面 |
| **ORM** | Prisma | 类型安全、开发体验好 |
| **消息队列** | Bull (基于Redis) | 轻量级、功能完善 |
| **文件存储** | 本地文件系统 + S3兼容 | 灵活部署、成本可控 |
| **容器化** | Docker + Docker Compose | 标准化部署、环境一致 |

### 前端技术栈
| 组件 | 技术选择 | 理由 |
|------|----------|------|
| **Web框架** | React 18+ | 生态丰富、性能优秀 |
| **语言** | TypeScript 5+ | 类型安全、开发效率高 |
| **状态管理** | Zustand | 轻量级、简单易用 |
| **UI框架** | Tailwind CSS + Headless UI | 灵活定制、无障碍支持 |
| **构建工具** | Vite | 开发体验好、构建速度快 |
| **桌面端** | Tauri (Rust + WebView) | 性能好、包体积小 |
| **移动端** | React Native | 跨平台、开发效率高 |

### 基础设施
| 组件 | 技术选择 | 理由 |
|------|----------|------|
| **CI/CD** | GitHub Actions | 集成性好、免费额度足 |
| **监控** | Prometheus + Grafana | 开源标准、功能强大 |
| **日志** | Winston + ELK Stack | 结构化日志、查询方便 |

## 📡 通信协议设计

### 1. WebSocket 协议
```typescript
// 连接建立
ws://{host}:{port}/ws?token={jwt_token}

// 消息格式
interface WebSocketMessage {
  id: string;           // 消息ID (UUID)
  type: MessageType;    // 消息类型
  payload: any;         // 消息内容
  timestamp: number;    // 时间戳
  sender: UserInfo;     // 发送者信息
}

// 消息类型
enum MessageType {
  TEXT = 'text',        // 文字消息
  FILE = 'file',        // 文件消息
  MEDIA = 'media',      // 多媒体消息
  TYPING = 'typing',    // 正在输入
  READ_RECEIPT = 'read', // 已读回执
  PRESENCE = 'presence', // 在线状态
}
```

## 🗃️ 数据模型设计

### 核心实体关系
```prisma
// Prisma 数据模型
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  username  String   @unique
  avatar    String?
  status    UserStatus @default(OFFLINE)
  
  // 关系
  messages  Message[]
  groups    GroupMember[]
  contacts  Contact[]
  
  // 时间戳
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model Message {
  id        String   @id @default(uuid())
  content   String?
  type      MessageType
  status    MessageStatus @default(SENT)
  
  // 关系
  sender    User     @relation(fields: [senderId], references: [id])
  senderId  String
  receiver  User?    @relation(fields: [receiverId], references: [id])
  receiverId String?
  group     Group?   @relation(fields: [groupId], references: [id])
  groupId   String?
  
  // 文件关联
  files     FileAttachment[]
  
  // 时间戳
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model FileAttachment {
  id        String   @id @default(uuid())
  name      String
  url       String
  size      Int
  mimeType  String
  
  // 关系
  message   Message @relation(fields: [messageId], references: [id])
  messageId String
  
  // 时间戳
  createdAt DateTime @default(now())
}
```

## 🚀 部署架构

### 开发环境
```yaml
# docker-compose.dev.yml
version: '3.8'
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: santi_dev
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: dev123
  
  redis:
    image: redis:7-alpine
  
  santi-server:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://dev:dev123@postgres:5432/santi_dev
      REDIS_URL: redis://redis:6379
```

### 生产环境
```yaml
# kubernetes 部署
apiVersion: apps/v1
kind: Deployment
metadata:
  name: santi-server
spec:
  replicas: 3
  selector:
    matchLabels:
      app: santi-server
  template:
    metadata:
      labels:
        app: santi-server
    spec:
      containers:
      - name: santi-server
        image: santi-channel:latest
        ports:
        - containerPort: 3000
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: santi-secrets
```
